    **buildFileTree(root: string): Promise<FileNode[]>**
    - Use `readdir` from `fs/promises` with `{ withFileTypes: true }`
    - Skip: entries starting with '.' EXCEPT '.env' (developers need to see .env in tree)
    - Skip directories in SKIP_DIRS = new Set(['node_modules', '.git', 'dist', '.next', '__pycache__', 'coverage'])
    - Add max-depth guard: accept optional `depth = 0` param; if depth >= 8, return [] (prevents hang on deep repos)
    - For directories: recurse with depth + 1; for files: add leaf node
    - Sort: directories before files, then alphabetically within each group
    - Return array of FileNode (the exported interface)

    **getGitChangedPaths(cwd: string): Promise<string[]>**
    - Use execFileAsync (import execFile from 'child_process' + promisify from 'util')
    - Run: execFileAsync('git', ['status', '--porcelain'], { cwd })
    - Parse porcelain v1 format: each non-empty line has "XY relpath" — relpath starts at index 3
    - Handle renamed files (porcelain shows "old -> new" or "R old\0new"): only take the portion after ' -> ' if present, or the last token for 'R' status lines
    - Strip surrounding double-quotes from paths with spaces (git wraps them)
    - Return array of absolute paths: path.join(cwd, relPath) for each
    - Wrap in try/catch — return [] if git not found or not a git repo (error is swallowed)

    Exports: FileNode interface, buildFileTree, getGitChangedPaths

  <action>
    Add three new routes to server/index.ts. Insert them between the existing /api/homedir route and the `attachWebSocketServer` call.

    Also add at the top: `npm install isbinaryfile` (run this before editing — the /api/file route needs it).

    **New imports to add at top of server/index.ts:**
    ```typescript
    import { readFile } from 'fs/promises';
    import { isBinaryFile } from 'isbinaryfile';
    import { buildFileTree, getGitChangedPaths } from './file-api.js';
    ```

    **Route 1 — GET /api/files:**
    - Query params: cwd (required string)
    - If cwd missing: return 400 { error: 'cwd required' }
    - Call buildFileTree(path.resolve(cwd))
    - Return 200 { tree: FileNode[] }
    - Try/catch: return 500 { error: String(err) }

    **Route 2 — GET /api/git-status:**
    - Query params: cwd (required string)
    - If cwd missing: return 400 { error: 'cwd required' }
    - Call getGitChangedPaths(path.resolve(cwd))
    - Return 200 { changed: string[] } (absolute paths)
    - No try/catch needed — getGitChangedPaths always returns [] on error

    **Route 3 — GET /api/file:**
    - Query params: cwd (required string), path (required string — relative path within cwd)
    - Security check FIRST: compute absPath = path.resolve(cwd, relPath); resolvedCwd = path.resolve(cwd); if !absPath.startsWith(resolvedCwd + '/') && absPath !== resolvedCwd → return 403 { error: 'Path outside cwd' }
    - Read file: buffer = await readFile(absPath)
    - Binary detection: binary = await isBinaryFile(buffer, buffer.length)
    - If binary: check ext against ['.png', '.jpg', '.jpeg', '.gif', '.webp', '.svg']; return { type: 'binary', isImage: boolean, base64: buffer.toString('base64'), ext }
    - If text: return { type: 'text', content: buffer.toString('utf-8') }
    - Try/catch: 404 if file not found (err.code === 'ENOENT'), 500 for other errors

    Use `const { cwd, path: relPath } = req.query as { cwd: string; path: string };` to avoid shadowing the `path` module.
  </action>
