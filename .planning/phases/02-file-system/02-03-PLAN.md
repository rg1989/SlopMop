  <files>client/components/FileTree.tsx, client/hooks/useFileTree.ts</files>
  <behavior>
    useFileTree:
    - When cwd changes (non-null), fetches /api/files?cwd=... and sets tree state
    - When cwd is null, tree is []
    - setMode('changes') fetches /api/git-status?cwd=... and sets changedPaths state as Set<string>
    - setMode('all') does NOT re-fetch git-status; changedPaths remains but mode is 'all'
    - Returns: { tree, changedPaths, mode, setMode }

    FileTree component:
    - Renders an unordered list of FileNode tree; directories rendered before files (server already sorts)
    - Each directory has a click handler that toggles open/closed state (initially open=true)
    - When collapsed, directory's children are not rendered in DOM
    - Files with path in changedPaths Set get class ft-changed on their element
    - Single-clicking a file calls onPreview(node.path)
    - Double-clicking a file calls onSelect(node.path)
    - Selected files (path in `selected` Set) get class ft-selected
    - When mode='changes': only render file nodes whose path is in changedPaths (directory nodes always rendered to provide structure); if a dir has no changed children after filter it can be omitted
  </behavior>
  <action>
    Create client/hooks/useFileTree.ts:
    - Import useState, useEffect, useCallback from react
    - Import FileNode from '../components/FileTree' (forward reference — define FileTree first, or define FileNode in a shared types file; simplest: define FileNode in useFileTree.ts and re-export, OR define in FileTree.tsx and import here)
    - Simplest approach: define FileNode interface in client/components/FileTree.tsx (exported), import in useFileTree.ts
    - Hook signature: export function useFileTree(cwd: string | null)
    - State: tree (FileNode[]), changedPaths (Set<string>), mode ('all' | 'changes')
    - useEffect on [cwd]: if cwd is null reset tree to []; else fetch `/api/files?cwd=${encodeURIComponent(cwd)}`, parse json, setTree(data.tree)
    - loadChanges (useCallback on [cwd]): fetch `/api/git-status?cwd=${encodeURIComponent(cwd!)}`, parse json, setChangedPaths(new Set(data.changed))
    - useEffect on [mode, loadChanges]: if mode === 'changes' call loadChanges()
    - Return { tree, changedPaths, mode, setMode }

    Create client/components/FileTree.tsx:
    - Export FileNode interface (same as server's — duplicate is fine, no shared TS between client and server at runtime)
    - Props interface for FileTree: nodes, selected (Set<string>), onSelect, onPreview, changedPaths (Set<string>), mode ('all' | 'changes')
    - Internal FileTreeNode component (not exported) handles single node recursion
    - Directory nodes: useState(true) for open; click toggles; render children when open
    - File nodes: className includes 'ft-selected' if selected.has(node.path), 'ft-changed' if changedPaths.has(node.path)
    - onClick → onPreview(node.path), onDoubleClick → onSelect(node.path)
    - In 'changes' mode: for file nodes, only render if changedPaths.has(node.path); for dir nodes, always render (structure preserved) but pass mode/changedPaths down
    - Export: FileTree (default not needed), FileNode
    - CSS classes to use: ft-tree (root ul), ft-dir, ft-dir-header, ft-caret (▾/▸), ft-file, ft-selected, ft-changed, ft-name
