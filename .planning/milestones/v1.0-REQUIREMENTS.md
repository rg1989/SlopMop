## v1 Requirements — Final Status

### Terminal

- [x] **TERM-01**: User can select a folder from the UI which opens a real PTY Claude CLI session in that directory — **Validated** (Phase 1)
- [x] **TERM-02**: Terminal renders full ANSI/color output with correct Claude CLI interactive behavior — **Validated** (Phase 1)
- [x] **TERM-03**: User can compose multiline messages in an input area before sending to the terminal — **Validated** (Phase 1)
- [x] **TERM-04**: Terminal supports scrollback buffer and keyboard copy/paste shortcuts — **Validated** (Phase 1)
- [x] **TERM-05**: Terminal resizes correctly when browser window or sidebar width changes — **Validated** (Phase 1)

### Files

- [x] **FILE-01**: User sees a VSCode-style collapsible file tree sidebar for the selected folder — **Validated** (Phase 2)
- [x] **FILE-02**: User can toggle the file tree between "All Files" and "Changes Only" (git-changed files) — **Validated** (Phase 2)
- [x] **FILE-03**: User can select files to attach to a message, with image/text previews shown before sending — **Validated** (Phase 2)
- [x] **FILE-04**: Attached files are passed to Claude CLI using @path syntax when the message is sent — **Validated** (Phase 2)
- [x] **FILE-05**: User can click a file in the explorer to preview its contents in a side panel — **Validated** (Phase 2)

### Voice Input

- [x] **VOICE-01**: User can record a voice message that is transcribed and dropped into the message input composer — **Validated** (Phase 3)
- [x] **VOICE-02**: Transcription is triggered by a button press (push-to-talk or start/stop toggle) — **Validated** (Phase 3)

### TTS Output

- [x] **TTS-01**: User can toggle a TTS mode where agent responses are streamed and read aloud using the AudioContext pattern — **Validated** (Phase 3)
- [x] **TTS-02**: TTS playback can be stopped at any point via a manual stop button — **Validated** (Phase 3)
- [x] **TTS-03**: User can interrupt TTS mid-sentence by speaking — playback stops and the transcribed speech is sent as a new message — **Validated** (Phase 3)
- [x] **TTS-04**: TTS and voice input are mutually exclusive — TTS pauses/stops when voice recording starts — **Validated** (Phase 3)

### Multi-Session

- [x] **SESS-01**: User can spawn multiple terminal sessions within the same workspace (soft cap of 8) — **Validated** (Phase 4)
- [x] **SESS-02**: User can navigate between sessions via a tab bar above the terminal area — **Validated** (Phase 4)
- [x] **SESS-03**: Each session tab shows a live status indicator (connecting / waiting / working / done / error) — **Validated** (Phase 4)
- [x] **SESS-04**: Session name is derived from the first prompt the user sends in that session — **Validated** (Phase 4)
- [x] **SESS-05**: Closed session metadata (name, status, cwd, timestamps) persists to localStorage and survives page reload — **Validated** (Phase 4)
- [x] **SESS-06**: Protocol and server architecture are structured for future PTY reconnect — stable sessionId carried from the start message — **Validated** (Phase 4)
