# Milestone v1.0: Foundation

**Status:** ✅ SHIPPED 2026-05-02
**Phases:** 1–9
**Total Plans:** 25 (phases 1–6 formal; phases 7–9 ad-hoc)
**Timeline:** 2026-04-30 → 2026-05-02 (3 days)

## Overview

v1.0 Foundation delivered a fully functional browser-based Claude CLI interface. The core loop: open a project folder, launch Claude CLI in a real PTY terminal, attach files, speak messages, hear responses, manage multiple sessions, and have all config/state persisted to disk. Phases 7–9 were shipped ad-hoc after the core 6-phase plan completed — Shiki highlighting, AI Guardian alignment rules, and UI state persistence round out the release.

## Milestone Summary

**Key Decisions:**
- Local web app over Electron: faster to ship, no packaging overhead, ghost-pepper is also local web
- Real PTY over chat-style input: full terminal fidelity — color, interactive prompts, stdin/stdout all work
- Files via CLI @path syntax: mirrors how Claude CLI actually works; no reimplementation needed
- Ghost-pepper AudioContext pattern for TTS: already solved streaming + interruption
- `.slop/` per-project config over localStorage: survives browser clears, shareable, version-controllable
- Propose+confirm for AI write-back: anything that influences future agent behavior requires explicit approval (AlterSquare Write Gate pattern)
- Modal anchor-to-top (`flex-start` + padding): header stays at fixed screen position regardless of content height

**Issues Resolved:**
- Double-spawn bug (PTY process created twice on folder select)
- Roadmap parser regression (parseRoadmapMd failing on ad-hoc phases)
- Duplicate CSS rules silently capping modal width at 480px
- Vault button/text collision on narrow panels

**Technical Debt Carried Forward:**
- PTY reconnect deferred to v1.1 — sessionId architecture is ready (SESS-06), live reconnect after browser reload not yet implemented
- Voice I/O (Phase 3) uses Web Speech API — upgrade to Whisper.js deferred to v2 (POW-04)
- No test coverage for ad-hoc phases 7–9
